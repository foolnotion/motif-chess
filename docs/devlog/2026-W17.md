# Week 17 — 2026-04-25

## Decisions

- **[D018] HTTP server as a separate `motif_http` binary:** The HTTP API layer runs as its own executable (`motif_http_server`) linked against `motif_http` (static lib), `motif_db`, `motif_search`, and cpp-httplib. The server uses pImpl to confine httplib's platform headers to `server.cpp`. `motif_search` is a PRIVATE dependency of `motif_http` — the HTTP module never reaches into `position_store` directly.

- **[D019] DB-level pagination, not HTTP-layer slicing:** A common opening position can have millions of matches. Materializing all rows before slicing at the HTTP layer would blow RAM and violate NFR01 (100ms P99). `LIMIT/OFFSET` lives in the DuckDB query so only the requested page is ever materialized. Default limit is 100 rows (~5 KB JSON); max clamped to 500 (~25 KB).

- **[D020] `ORDER BY game_id, ply` for stable pagination:** Without an explicit ORDER BY, DuckDB can return rows in any physical order across pages, causing duplicates or gaps. Added stable ordering to `query_by_zobrist` whenever `LIMIT`/`OFFSET` is present.

- **[D021] `limit=0` treated as "return empty array":** The story spec says "at most `limit` items are returned". `limit=0` returning all rows was a footgun. Changed the endpoint to short-circuit: if `limit` is explicitly `0`, return `[]` immediately without querying.

- **[D022] `perf-release` preset committed to project presets:** Moved from `.gitignore`d `CMakeUserPresets.json` into `CMakePresets.json`. The preset is a project concern — every developer and CI needs it to run performance tests. `CMakeUserPresets.json` reduced to an empty shell for machine-specific overrides only.

- **[D023] `httplib` added to Nix dev shell:** The `flake.nix` package already listed `httplib` in `buildInputs`, but the dev shell only inherited `nativeBuildInputs` + `buildInputs` and missed propagating httplib's CMake config. Fresh configure presets like `perf-release` couldn't find it. Added `httplib` explicitly to the dev shell's `buildInputs`.

## Learned

- **cpp-httplib route pattern `:param` does not match empty segments.** `/api/positions/:zobrist_hash` does not fire for `/api/positions/` — an empty path segment. The empty-hash AC4 case required separate explicit routes for `/api/positions` and `/api/positions/`.

- **`std::from_chars` with `std::size_t` naturally rejects negative values** on platforms where `size_t` is unsigned — the parse fails, which is exactly what AC5 requires for negative `offset`/`limit` parameters.

- **DuckDB `LIMIT 0` means "return zero rows"**, so the pagination guard must be `limit > 0` to append the clause. When `limit == 0` (the default "no pagination" value), the query returns all matching rows — which is correct for internal callers like `opening_stats` but wrong for the HTTP endpoint where `?limit=0` should return `[]`.

- **Glaze reflection requires named namespaces.** Anonymous-namespace structs don't have external linkage, so `glz::write_json` can't find their metadata. The `health_response` and `error_response` structs must live in `motif::http::detail`.

## Problems and Solutions

- **Problem:** `limit=0` in the endpoint bypassed pagination and returned all matches.
  **Solution:** Added a guard in the route handler: if the client explicitly passed `limit=0`, return `[]` immediately.

- **Problem:** Paginated queries had no `ORDER BY`, so `LIMIT/OFFSET` pages could duplicate or skip rows across requests.
  **Solution:** Appended `ORDER BY game_id, ply` to the DuckDB query when `limit > 0`.

- **Problem:** `/api/positions/` (empty hash) didn't match the `:zobrist_hash` route parameter, returning 404 instead of 400.
  **Solution:** Registered explicit handlers for `/api/positions` and `/api/positions/` that return the AC4 error.

- **Problem:** `perf-release` CMake preset was `.gitignore`d, so new clones lacked it and CI couldn't run performance tests.
  **Solution:** Moved all project-level presets into `CMakePresets.json`; reduced `CMakeUserPresets.json` to a stub.

- **Problem:** Fresh CMake configures (e.g., `perf-release`) failed to find `httplib` because the Nix dev shell didn't propagate its CMake config.
  **Solution:** Added `httplib` to `buildInputs` in the dev shell section of `flake.nix`.
